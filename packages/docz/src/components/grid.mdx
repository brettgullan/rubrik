<lede>A responsive Grid component, enabling complex responsive grids using simple style props.</lede>  

The Grid component is intended to solve a number of issues that are not easily handled by conventional Flexbox approaches:

* Provide fluid, responsive grid layouts with fixed gutters;
* Support variable grid item spans;
* Use a declarative, props-based configuration;
* Declare widths and margins directly on Grid items — i.e. no wrappers.
* Don't use negative margins, all margins and sizing remain strictly within the layout container element.

Grid uses CSS flex and the calc() function. No CSS Grids. In order to maintain maximum browser compatibility.

## Usage

The Grid component uses two main props — `columns` and `gutters`. If using responsive array format, both props _must_ have the same number of values declared.

In addition, each top-level child component may optionally include a `span` prop. This also accepts a responsive array format, and _must_ also have the same number of elements as the `columns` and `gutters` props.

In the example below, the Grid layout changes across four breakpoints, with elements 2 - 5 spanning different column counts across each breakpoint step.

Note that if the total span of adjacent elements exceeds the number of columns in a row, the "excess" elements automatically wrap to the next row.

    columns={[1, 2, 2, 3]}
